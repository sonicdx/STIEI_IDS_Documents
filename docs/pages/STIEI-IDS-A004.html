学院微信使用域名集合：

  • wchattest.stiei.edu.cn 测试部署域名
  • wchat.stiei.edu.cn 正式部署域名

附加： 所有域名下需要能配置一个可以访问到的静态文件，用于存放微信服务的认证文件。具体的文件名称与内容由微信公众号后端生成与决定。

 

学院微信应用部署方式的前期调研

​ 由于学院信息化建设的需要，拓展移动应用平台的部署已是当务之急。而要建设移动应用平台，则首推微信公众号（企业号）的应用开发。在建设与部署学院的微信号应用过程中，我们信息化小组发现。由于微信公众号的平台限制，一个公众号只允许绑定最多3个安全域名。而实际需要建设的微信应用数量则远超该数字。

​ 解决路径主要有2条：增加微信公众号或者通过反向代理。

​ 增加微信号的方式目前暂时被搁置，因为虽然该路径最简单。但是在后期需要大量的申请微信公众号与相关域名。这样，在财务管理与业务流程上会增加许多工作量与不确定性。而且在技术上也会为微信平台上，统一用户身份的确认增加了新的技术阻碍。

​ 目前经过信息化小组的内部讨论，决定采用反向代理的方式，实现多个微信Web应用共享同一个微信域名的技术方案。

 

学院微信应用部署原则

​ 上海电子信息职业技术学院微信应用部署的基本原则如下：

  • 所有微信Web应用都部署内部服务区域内，原则上不直接对外公开。
  • 微信Web应用在内部部署后的访问方式，应统一为 <IP>:<Port> 的形式
  • 学院需要为每个微信Web应用提供一个唯一的应用 ID 与 应用短名称，该 应用清单 需由学院信息化建设小组制定，并由图文信息中心保存留档。
  • 图文信息中文 需要依据 应用清单 ，为各个微信Web应用建立正确的反向代理规则与代理行为。

 

反向代理规则与代理行为说明

  1. 在测试期间的所有微信Web应用部署在 wchattest.stiei.edu.cn 的测试域名下。
  2. 正式运行的微信Web应用部署在 wchat.stiei.edu.cn 的正式域名下。
  3. 所有微信Web应用将以所部署域名下的二级目录的方式对外网公开。二级目录的名称等同于 应用清单 里的应用短名称 。
  4. 代理服务器在反向代理的过程中，需要为Http请求添加一个名为“X-Forwarded-For”的Http头部，该头部的内容为该微信Web应用的应用短名称 。
